## 🔍 發現的問題

### 1. 會議室重複問題 (高優先級)
**問題**: ID 1042 的會議室有重複（同一會議室出現多次）

**範例**:
```json
{
  "name": "1F 前瞻廳",
  "floor": "1F",
  "capacity": "220",
  "equipment": "投影機, 麥克風, 音響, 螢幕, 白板"
}
```
這個會議室在 rooms 陣列中出現了 4 次。

**影響**:
- 實際會議室數量可能被高估
- 資料不乾淨

**建議修正**:
- 實作會議室去重邏輯
- 使用 `name + floor` 作為唯一鍵

---

### 2. HTML 標籤未清理 (中優先級)
**問題**: 擷取的文字包含 HTML 標籤

**範例**:
```json
{
  "rules": {
    "catering": "<a href=\"/1122/2141/3158/NPmeetingVenue\" target=\"_self\" title=\"餐飲服務\">餐飲服務</a>",
    "decoration": "<li class=\"container-single\"><a href=\"./download\">下載專區(主辦單位/裝潢商)</a></li>"
  },
  "accessInfo": {
    "mrt": "<meta name=\"description\" content=\"台北六福萬怡酒店位於南港車站上方..."
  }
}
```

**建議修正**:
- 使用 BeautifulSoup 的 `.get_text()` 清理 HTML
- 移除 `<meta>`, `<link>` 等 meta 標籤

---

### 3. 設備欄位經常為空 (中優先級)
**問題**: 很多會議室的 equipment 欄位為空

**統計**:
- ID 1042: 前 4 個會議室有設備，後面 42 個都沒有
- ID 1043: 所有會議室都沒有設備資訊
- ID 1049: 所有會議室都沒有設備資訊

**可能原因**:
1. 會議室頁面沒有列出設備
2. 設備資訊在另一個頁面
3. 需要更深入的頁面爬取

---

### 4. 容量格式不統一 (低優先級)
**問題**: 有些會議室有容量，有些沒有

**範例**:
```json
// 有容量
{"name": "1F 前瞻廳", "capacity": "220"}

// 沒有容量
{"name": "第一會議室", "capacity": ""}
```

**建議修正**:
- 標準化容量欄位
- 空值設為 null 而非空字串

---

### 5. 平面圖擷取率低 (低優先級)
**問題**: 只有 4 個場地 (10%) 有平面圖資料

**可能原因**:
1. 大多數場地沒有提供線上平面圖
2. 平面圖在 PDF 檔案中
3. 需要特殊方式擷取

# 場地爬蟲解析邏輯修正報告

## 問題診斷

### 原始錯誤
爬蟲將晶華酒店容量表格的 **776 / 8508** 錯誤解析：
- ❌ 將 8508 (sqft) 當成 sqm
- ❌ 將面積數值混淆為容量
- ❌ 缺失尺寸資訊（長、寬、高）

### 正確理解
```
貴賓廳 | 776 / 8508 | 2.3 | 580 | 576 | 700 | - | 429 | - | -
         ↓    ↓       ↓    ↓     ↓     ↓          ↓
       sqm  sqft   高   前台  宴會  劇院       課堂
```

## 修正方案

### 1. 正確解析雙單位尺寸

```python
def _parse_size_dual(self, text: str):
    """解析 776 / 8508 格式 (sqm / sqft)"""
    match = re.match(r'(\d+(?:\.\d+)?)\s*/\s*(\d+)', text)
    if match:
        return float(match.group(1)), int(match.group(2))
    return None, None
```

**結果**：776 sqm（而非錯誤的 8508）

### 2. 正確提取容量資訊

### 3. 正確計算坪數


## 技術要點

### 關鍵發現
1. **單位混淆是主要錯誤來源** - sqft/sqm 混淆導致數值錯誤
2. **表格格式識別很重要** - 不同飯店有不同格式
3. **容量欄位順序** - 前台、宴會、劇院、課堂等順序需正確對應

### 解析模式
```python
# 模式 1: 雙單位尺寸
776 / 8508  →  (sqm, sqft)

# 模式 2: 容量表格

# 模式 3: 尺寸文字
31.8 × 24.4 × 2.3 公尺
```

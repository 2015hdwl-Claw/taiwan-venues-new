### 1. 維多利亞酒店（ID: 1122）

**狀態**: 手動完成，價格覆蓋 50%

**根本原因**:
- ❌ **PDF 資料不完整** - 官方 PDF 只包含部分會議室的價格
- ❌ **網頁無價格資訊** - 官網會議室頁面沒有顯示價格
- ⚠️ **需要聯絡詢問** - 價格需要諮詢才能取得

**具體情況**:
```
PDF 來源: victoria_capacity.pdf
包含內容: 容量、面積
缺少內容: 價格（只有部分會議室）
```

**為什麼無法自動取得價格**:
1. 官網會議室頁面: https://www.grandvictoria.com.tw/meeting-rooms
2. 頁面顯示: 會議室名稱、容量、面積、照片
3. 頁面**沒有顯示價格**（需要填寫表單詢問）
4. 價格通常根據時段、套餐而定

**結果**: 13 個會議室中，只有 6-7 個從其他來源找到價格

### 2. 台北世貿中心（ID: 1049）

**狀態**: 容量 100%，價格/設備 0%

**根本原因**:
- ❌ **會議室頁面沒有價格** - 每個會議室頁面只顯示容量

- ❌ **沒有面積資訊** - 頁面上沒有坪數資訊
- ⚠️ **需要查看 PDF 或詢問** - 價格可能在 PDF 中或需要詢問

**具體情況**:
```
會議室頁面範例: https://www.twtc.com.tw/meeting1
頁面內容:
  ✅ 會議室名稱: "第一會議室"
  ✅ 容量: "84人"
  ❌ 價格: 無
  ❌ 設備: 無
  ❌ 面積: 無
```

**為什麼無法取得價格/設備**:
1. 世貿中心的會議室頁面**非常簡潔**
2. 只顯示基本資訊（名稱、容量）
3. 價格可能需要:
   - 下載會議室租借規範 PDF
   - 或發送詢問表單

**結果**: 7 個會議室都有容量，但完全沒有價格/設備資料

### 3. 台北萬豪酒店（ID: 1103）

**狀態**: 容量 100%，價格/設備 0%

**根本原因**:
- ❌ **JavaScript 動態載入** - 會議室資料由 JavaScript 動態生成
- ❌ **需要 API 調用** - 價格/設備資料在 API 端點中
- ❌ **BeautifulSoup 無法提取** - 靜態爬蟲無法取得動態內容
- ⚠️ **需要 Playwright** - 必須使用瀏覽器自動化工具

**具體情況**:
```
URL: https://www.taipeimarriott.com.tw/meetings/events
頁面特徵:
  ✅ 有會議室列表
  ✅ 有容量資訊
  ❌ 價格/設備由 JS 動態載入
  ❌ 靜態 HTML 中沒有這些資料
```

**為什麼靜態爬蟲失敗**:
1. 萬豪酒店使用 **React/Angular 等 SPA 框架**
2. 初始 HTML 只是空殼 `<div id="root"></div>`
3. 真實資料由 JavaScript 從 API 取得:
   ```
   GET /api/meetings/rooms
   Response: { rooms: [{ name: "...", price: ..., equipment: ... }] }
   ```
4. BeautifulSoup 只能讀取初始 HTML，看不到 API 回應

**結果**: 7 個會議室有容量，但價格/設備在 API 中（未被提取）

- ❌ **部分會議室資料缺失** - 23 個會議室中 3 個無容量

**具體情況**:
```
URL: https://www.brotherhotel.com.tw/?cat=75
頁面特徵:
  ✅ 有會議室列表
  ⚠️ 容量資訊不完整（20/23）
  ❌ 價格/設備由 JS 動態載入
```

**為什麼容量資訊不完整**:
1. 會議室頁面可能使用 **分頁載入**
2. 需要點擊"下一頁"或滾動頁面
3. 靜態爬蟲只能看到第一頁的 20 個會議室
4. 另外 3 個會議室在後續頁面中

**結果**: 23 個會議室中 20 個有容量，但價格/設備完全缺失

### 6. 台北國際會議中心 TICC（ID: 1448）

**錯誤**: 403 Forbidden

**根本原因**:
- ❌ **伺服器拒絕訪問**
- ❌ **需要特定 User-Agent**
- ❌ **需要從台灣 IP 訪問**
- ❌ **需要 Cookie 或 Session**
- ❌ **可能有反爬蟲機制**

**具體情況**:
```
URL: https://www.ticc.com.tw/
HTTP 回應: 403 Forbidden

可能原因:
  1. 檢測到爬蟲 User-Agent
  2. 只允許台灣 IP 訪問
  3. 需要先訪問首頁建立 Session
  4. 有 Cloudflare 或 WAF 保護
```

**為什麼會 403**:
1. TICC 網站可能有 **Cloudflare 保護**
2. 檢測到自動化工具的特徵:
   - Python requests 的預設 User-Agent
   - 沒有 JavaScript 執行能力
   - 請求頻率過快
3. 解決方案:
   - 使用真實瀏覽器 User-Agent
   - 使用 Playwright（模擬真實瀏覽器）
   - 使用台灣 VPN

### 8. 青青婚宴會館（ID: 1129）

**狀態**: 10 個宴宴廳，容量/價格/設備 0%

**根本原因**:
- ❌ **婚宴場地的特殊性** - 價格依方案、日期、人數而定
- ❌ **資料在圖片或 PDF 中** - 無法用文字爬取
- ❌ **需要諮詢** - 官網鼓勵填寫表單詢問
- ❌ **容量可能不是重點** - 婚宴更重視桌數而非人數

**具體情況**:
```
URL: https://www.77-67.com/
宴宴廳頁面範例: /champs-hall
頁面內容:
  ✅ 宴宴廳照片（精美）
  ✅ 特色描述（浪漫、豪華...）
  ✅ 聯絡資訊
  ❌ 容量（人數）
  ❌ 價格
  ❌ 設備清單
```

**為什麼沒有容量/價格**:
1. **婚宴場地的價格結構複雜**:
   - 依平日/假日不同
   - 依午宴/晚宴不同
   - 依桌數（10桌/20桌/30桌）不同
   - 依套餐方案不同
2. **官網不顯示價格**:
   - 鼓勵聯絡婚宴顧問
   - 價格可能經常調整
   - 需要根據需求報價
3. **容量用桌數表示**:
   - "香榭廳: 20-30桌"
   - 而非 "香榭廳: 200-300人"

**結果**: 10 個宴宴廳都有名稱和 URL，但完全沒有數值化資料

## 問題類型總結

### 類型 1: JavaScript 動態載入（2 個）

**場地**: 台北萬豪、台北兄弟大飯店

**特徵**:
- 使用 SPA 框架（React/Vue/Angular）
- 資料在 API 端點中
- 靜態爬蟲無法提取

**解決方案**:
```python
# 使用 Playwright
from playwright.sync_api import sync_playwright

with sync_playwright() as p:
    browser = p.chromium.launch()
    page = browser.new_page()
    page.goto(url)
    # 等待 JS 載入完成
    page.wait_for_selector('.room-data')
    # 提取資料
    data = page.evaluate('() => window.roomData')
    browser.close()
```

## 技術限制分析

### 為什麼自動化爬蟲無法解決所有問題？

| 問題類型 | 場地數 | 自動化解決方案 | 成功率 |

| 連線失敗 | 3 | 確認 URL / VPN | ⚠️ 50% |

**結論**:
- ✅ **20%** 的場地可以用簡單爬蟲解決（靜態 HTML）
- ⚠️ **60%** 的場地需要進階工具（Playwright、PDF 解析）
- ❌ **20%** 的場地需要手動處理（連線失敗、需詢問）

# 資料驗證工具使用指南

## 📋 概述

本專案已建立完整的資料驗證系統，確保所有場地資料在匯入前都經過嚴格驗證。

**核心原則：資料在源頭就應該正確，不是事後清理**

---

## 📁 相關文件

1. **DATA_EXTRACTION_SOP.md** - 完整的資料擷取標準作業程序
2. **validate_venue_data.py** - 自動驗證腳本
3. **quality_report.json** - 最新品質報告（自動生成）

---

## 🚀 快速開始

### 1. 驗證單一場地

```bash
python3 validate_venue_data.py venue.json
```

### 2. 驗證所有場地

```bash
python3 validate_venue_data.py --all venues.json
```

### 3. 生成品質報告

```bash
python3 validate_venue_data.py --report venues.json --output quality_report.json
```

### 4. 只檢查必填欄位

```bash
python3 validate_venue_data.py --check-required venue.json
```

### 5. 只計算品質評分

```bash
python3 validate_venue_data.py --quality-score venue.json
```

## 📝 品質評分標準

### A 級（90-100 分）

- ✅ 所有必填欄位完整
- ✅ 格式驗證全部通過
- ✅ 官網驗證成功
- ✅ 有主照片和所有會議室照片
- ✅ 價格、容納人數等數值合理

### B 級（70-89 分）

- ✅ 所有必填欄位完整
- ✅ 格式驗證通過
- ⚠️ 官網驗證部分成功
- ⚠️ 部分會議室缺少照片
- ⚠️ 部分選填欄位空白

### C 級（<70 分）

- ❌ 缺少必填欄位
- ❌ 格式驗證失敗
- ❌ 官網無法驗證
- ❌ 缺少照片
- ❌ 資訊明顯錯誤

**處理方式**: 不建議匯入，需補充資料

## 🔧 整合到工作流程

### 新增場地時

1. 收集場地資料
2. 保存為 JSON 格式
3. 執行驗證腳本
4. 確認評分為 A 或 B
5. 才允許匯入系統

### 批量更新時

1. 準備更新資料
2. 執行批量驗證
3. 生成品質報告
4. 修正問題場地
5. 才允許匯入系統

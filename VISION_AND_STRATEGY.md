# 活動大師 EventMaster - 願景與戰略 2026

**制定日期**: 2026-03-24
**戰略定位**: AI時代的活動場地智能基礎設施
**核心策略**: API-first，成為AI模型查詢台灣場地的首選接口

---

## 🎯 願景聲明

### 核心使命
**成為AI時代台灣活動場地領域的權威知識圖譜與智能接口**

我們不是另一個場地預訂網站，而是**AI模型的場地智能合作夥伴**。

### 問題陳述

#### AI模型面臨的三大挑戰

1. **資料碎片化**
   - 場地資訊散落在數百個獨立官網
   - 資料格式不統一（坪數 vs 平方公尺、劇院式 vs 宴會式）
   - AI每次查詢需要耗時2-5秒爬取多個網站

2. **即時性缺失**
   - AI訓練資料是靜態的，無法知道今天哪個場地可訂
   - 價格變動、檔期狀態無法即時反映
   - 導致AI給出過時或錯誤的建議

3. **隱藏知識無法取得**
   - AI不知道「晶華酒店週末不得外叫餐飲」這類隱藏限制
   - 無法取得場地的pro tips（如「訂9:00-17:00比8:00-16:00便宜20%」）
   - 導致用戶依AI建議選擇後才發現不符合需求

#### 用戶的「繞圈」痛苦

```
傳統路徑：
用戶問AI → AI不知道細節 → AI再問用戶 → 用戶不知道專業名詞
→ AI給出建議 → 用戶發現不符合需求 → 重新開始
```

**平均繞圈3-5次才能找到合適場地**

### 我們的解決方案

#### EventMaster API-first架構

```
理想路徑：
用戶說「我要辦一個像TED的活動」
→ AI調用EventMaster API（單次請求<100ms）
→ API返回「TED風格需要：舞台高度、燈光、視聽設備、容納人數」
→ AI直接給出5個符合TED規格的場地選項
→ 包含隱藏知識（如「週末需提早3個月預訂」）
```

**一次查詢，精準命中**

## 關鍵發現：為什麼補強率 0%？

### 問題 1：假會議室（False Positives）

爬蟲發現的 141 個「會議室」中，大部分不是真實的會議室：

**範例 - 兄弟大飯店**:
```
❌ "即時消息餐飲專案住宿專案婚宴專案會議專案"（導航文字）
❌ "訂餐洽詢專線：(02)2712-3456 轉 花香廳(鐵板燒)"（包含電話號碼）
❌ "美味食旅菊花廳(日本料理)蝶花廳及酒廊..."（連結的餐廳名稱）
✅ "菊花廳"、"蝶花廳"、"梅花廳"（真實會議室，但缺少資料）
```

### 問題 2：資料在第三層，但第三層不存在

技術檢測顯示 7/10 場地的資料在「詳情頁」（第三層），但實際爬取後發現：

1. **沒有真正的詳情頁**: 連結指向的是 JavaScript `void(0)` 或同一頁的錨點
2. **資料格式非結構化**: 會議室資訊以段落文字呈現，非結構化資料
3. **關鍵資料缺失**: 容量、面積、價格完全未在 HTML 中出現

### 問題 3：資料可能在 PDF 或 JavaScript 中

**集思台大會議中心**（ID: 1128）是最佳範例：
- HTML 只有 4 個會議室
- 實際有 12 個會議室（含容量、面積、價格）
- **資料在 PDF 中**: `台大_場地租用申請表_20250401.pdf`

## 結論

### 三級爬蟲方法是否正確？

**✅ 是的，但需要改進**

三級爬蟲的架構是正確的：
1. ✅ 技術檢測（判斷網站類型）
2. ✅ 深度爬蟲（主頁 → 會議頁 → 詳情頁）
3. ✅ 驗證寫入（檢查資料完整性）

**但實作上有問題**：
- ❌ 過濾邏輯不夠嚴格（假會議室太多）
- ❌ 未支援 JavaScript 動態載入（需要 Playwright）
- ❌ PDF 提取未整合（只有發現，未解析）

### 現實情況

**自動化爬蟲無法解決所有問題**：

1. **部分場地資料根本不在網站上**（需要電話確認）
2. **JavaScript 動態載入需要特殊工具**（Playwright）
3. **PDF 價格表需要手動或 AI 輔助解析**

**最佳策略是混合方法**：
- 80% 自動化爬蟲（BeautifulSoup + Playwright）
- 15% PDF 解析
- 5% 手動確認

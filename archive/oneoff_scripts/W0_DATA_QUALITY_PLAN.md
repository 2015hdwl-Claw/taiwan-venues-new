### 階段 2：補充維度資訊（優先級 🟠）

#### 2.1 問題分析

**診斷結果**：
- 51/52 場地缺少 `totalArea` 和 `totalAreaUnit`
- 這是 AI 判斷場地大小的關鍵資訊

**影響**：
```python
# AI 應用無法回答：
「這個場地適合辦 500 人的活動嗎？」
→ 無維度資訊 → 無法判斷 → 推薦不準確
```

#### 2.2 修復策略

**來源優先順序**：
1. **官網 PDF 場地菜單**（最準確）
   - 例如：晶華酒店有完整的會議室坪數
2. **官網會議場地頁面**
   - 例如：國賓酒店有列出面積
3. **估算基於容量**
   - 劇院式 1 人 ≈ 1 平方公尺
   - 宴會式 1 人 ≈ 1.5 平方公尺

**修復批次**：
- 批次 1：台北 5 星級飯店（晶華、國賓、君悅、W 飯店）
- 批次 2：台北 4 星級飯店（六福、薇閣、麗緻等）
- 批次 3：其他城市飯店

### 階段 4：補充價格資訊（優先級 🟡）

**診斷結果**：
- 79 個會議室沒有價格

#### 4.2 修復策略

   - 參考同級飯店價格
   - 標註為「估計價格」

### 階段 5：建立驗證機制（優先級 🟢）

#### 5.1 自動化檢查腳本

創建 `scripts/validate_data.py`：

```python
#!/usr/bin/env python3
"""
venues.json 資料品質自動檢查
"""

import json
from typing import Dict, List

def validate_venues(data: List[Dict]) -> Dict:
    """檢查場地資料完整性"""

    issues = {
        'critical': [],  # 嚴重錯誤（阻礙上線）
        'warning': [],   # 警告（建議修正）
        'info': []       # 資訊（可選優化）
    }

    for venue in data:
        venue_id = venue.get('id')
        venue_name = venue.get('name')

        # 檢查嚴重錯誤
        if not venue.get('rooms'):
            issues['critical'].append({
                'venue_id': venue_id,
                'venue_name': venue_name,
                'issue': 'Missing rooms data'
            })

        # 檢查維度資訊
        if not venue.get('totalArea') and not venue.get('totalAreaUnit'):
            issues['warning'].append({
                'venue_id': venue_id,
                'venue_name': venue_name,
                'issue': 'Missing dimensions'
            })

        # 檢查會議室
        for idx, room in enumerate(venue.get('rooms', [])):
            if not room.get('images'):
                issues['warning'].append({
                    'venue_id': venue_id,
                    'venue_name': venue_name,
                    'room_name': room.get('name'),
                    'issue': 'Room has no images'
                })

            if not room.get('price'):
                issues['info'].append({
                    'venue_id': venue_id,
                    'venue_name': venue_name,
                    'room_name': room.get('name'),
                    'issue': 'Room has no price'
                })

    return issues

if __name__ == '__main__':
    with open('venues.json', 'r', encoding='utf-8') as f:
        data = json.load(f)

    issues = validate_venues(data)

    print(f'=== Data Quality Report ===')
    print(f'Critical issues: {len(issues["critical"])}')
    print(f'Warnings: {len(issues["warning"])}')
    print(f'Info: {len(issues["info"])}')

    # 輸出詳細報告
    with open('data_quality_report.json', 'w', encoding='utf-8') as f:
        json.dump(issues, f, ensure_ascii=False, indent=2)
```

#### 5.2 資料完整性評分系統

建立評分標準：

| 評分項目 | 滿分 | 達標標準 |
|---------|------|---------|
| 場地基本資訊 | 20 | 地址、電話、網址完整 |
| 會議室數量準確性 | 20 | 與官網一致 |
| 維度資訊 | 15 | 有 totalArea 或坪數 |
| 容量資訊 | 15 | 有各種佈局容量 |
| 視覺資產 | 15 | 每間會議室 ≥3 張照片 |
| 價格資訊 | 15 | 有明確價格或價格區間 |
| **總分** | **100** | **≥80 分為「可信賴」** |

**評級標準**：
- **A 級 (90-100)**：完整，可直接上線
- **B 級 (80-89)**：良好，次要資訊可補充
- **C 級 (70-79)**：尚可，需補充關鍵資訊
- **D 級 (60-69)**：不合格，需大量修正
- **F 級 (<60)**：嚴重不合格，需重新調查

## 🎯 為什麼這麼嚴格？

### 錯誤的代價

```
場景：AI 應用推薦場地給用戶

❌ 資料有問題：
「我推薦茹曦酒店，有 3 間會議室...」
→ 用戶去官網看 → 實際有 15+ 間
→ 用戶：「這個 AI 很不準確」
→ 流失用戶 + 損害信譽

✅ 資料準確：
「我推薦茹曦酒店，有 4 個主要場地，包含 15+ 間會議室...」
→ 用戶去官網看 → 確實如此
→ 用戶：「這個 AI 很可靠」
→ 獲得信任 + 持續使用
```

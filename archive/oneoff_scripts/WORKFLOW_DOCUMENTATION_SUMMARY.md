```
主要儲存:
  - venues.json (主資料庫)

備份儲存:
  - venues.json.backup.{timestamp}
  - removed_*.json (下架場地記錄)

報告儲存:
  - *_REPORT.md (各種處理報告)
  - *_SUMMARY.md (執行摘要)
```

### 爬蟲流程

```
┌─────────────────────────────────────────────────────┐
│                   爬蟲執行流程                        │
└─────────────────────────────────────────────────────┘

[輸入] 官網 URL 或 PDF URL
   │
   ▼
[選擇] 爬蟲工具
   │
   ├─ deep_scraper_v2.py     (完整六階段)
   ├─ full_site_scraper_v4.py  (全站爬取)
   └─ parallel_venue_scraper.py (快速批次)
   │
   ▼
[執行] 爬取資料
   │
   ├─ HTML 內容
   ├─ PDF 資料
   └─ 自動發現頁面
   │
   ▼
[驗證] 資料品質
   │
   ├─ 完整性檢查
   ├─ 準確性檢查
   └─ 一致性檢查
   │
   ▼
[更新] venues.json
   │
   ├─ 自動備份
   ├─ 更新資料
   └─ 標記來源
   │
   ▼
[輸出] 處理報告
```

### 驗證流程

```
┌─────────────────────────────────────────────────────┐
│                   驗證檢視流程                        │
└─────────────────────────────────────────────────────┘

[檢查] 資料完整性
   │
   ├─ check_data_quality.py    (品質檢查)
   ├─ check_venue_details.py   (詳情檢視)
   └─ check_db_status.py       (狀態檢查)
   │
   ▼
[問題] 發現問題
   │
   ├─ 資料缺失 → 補充資料
   ├─ 格式錯誤 → 修正格式
   └─ 準確性問題 → 重新驗證
   │
   ▼
[修正] 更新資料
   │
   ├─ update_specific_venues.py
   └─ 手動修正
   │
   ▼
[報告] 生成報告
```

## 🎓 核心概念

### 資料庫設計

**單一資料庫**: `venues.json`
- 優點: 簡單、易於維護、易於備份
- 格式: JSON
- 編碼: UTF-8
- 結構: 場地列表，每個場地包含完整資訊

**資料來源標記**:
- `metadata.source`: 資料來源 (official_pdf, html, manual)
- `metadata.scrapeVersion`: 爬蟲版本
- `metadata.scrapeConfidenceScore`: 信心分數 (0-100)

### 爬蟲策略

**三層策略**:
1. **完整爬蟲** (deep_scraper_v2.py)
   - 重要場地
   - 需要完整資料
   - 較慢但最完整

2. **全站爬蟲** (full_site_scraper_v4.py)
   - 一般場地
   - 自動發現頁面
   - 平衡速度與完整度

3. **快速爬蟲** (parallel_venue_scraper.py)
   - 批次更新
   - 大量場地
   - 速度優先

### 驗證機制

**三層驗證**:
1. **完整性**: 必備欄位檢查
2. **準確性**: 數值合理性檢查
3. **一致性**: 資料間邏輯檢查

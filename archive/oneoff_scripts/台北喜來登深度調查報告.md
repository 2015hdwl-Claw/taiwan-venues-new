## 🎯 根本原因分析

### 1. 為何會議室照片會被錯誤清空？

**直接原因**：
- `comprehensive_taipei_update.py` 腳本中的會議室模板設定 `"images": []`
- 腳本使用**完全替換**策略，而非**合併更新**

**技術細節**：
```python
# comprehensive_taipei_update.py 中的模板
TAIPEI_HOTEL_ROOMS = {
    1067: {  # 台北喜來登
        "rooms": [
            {
                "name": "福廳",
                "capacity": {...},
                "area": 200,
                "price": {...},
                "equipment": [...],
                "images": []  # ❌ 空陣列，覆蓋了原有照片
            },
            ...
        ]
    }
}

# 更新邏輯（推測）
venue["rooms"] = TAIPEI_HOTEL_ROOMS[venue_id]["rooms"]  # 完全替換
```

**應該怎麼做**：
```python
# 正確的合併更新策略
for room in venue["rooms"]:
    room_id = room.get("id")
    if room_id in update_data:
        # 只更新指定欄位，保留其他欄位
        for key, value in update_data[room_id].items():
            if value is not None:  # 只更新非空值
                room[key] = value
```

### 2. 為何空間資訊（長寬高）會被刪除？

**直接原因**：
- 批量更新腳本的模板中**沒有包含** floor, ceiling, hasWindow 等欄位
- 腳本設計時可能認為這些欄位「不重要」或「難以批量獲取」

**深層原因**：
- **缺乏資料欄位規範**：沒有明確定義會議室「必須」包含哪些欄位
- **批量更新的便利性 vs 資料完整性權衡**：選擇了便利性，犧牲了完整性
- **缺乏資料驗證機制**：更新後沒有檢查是否丟失重要欄位

**資料欄位重要性分級**：

| 級別 | 欄位 | 重要性 | 目前狀態 |
|------|------|--------|---------|
| 🔴 必要 | name, capacity, area | 場地基本資訊 | ✅ 存在 |
| 🟠 重要 | floor, ceiling, hasWindow | 空間規劃關鍵 | ❌ 缺失 |
| 🟡 建議 | features, notes | 決策輔助 | ❌ 缺失 |
| 🟢 可選 | pillar, shape | 補充資訊 | ✅ 存在 |

#### ✅ 建立資料驗證腳本

**建議**：建立 `validate_room_data.py`

```python
#!/usr/bin/env python3
"""會議室資料驗證腳本"""

import json
from typing import Dict, List

# 必要欄位
REQUIRED_FIELDS = ["id", "name", "capacity", "area"]
# 重要欄位（缺失時發出警告）
IMPORTANT_FIELDS = ["floor", "ceiling", "hasWindow", "images"]
# 建議欄位（缺失時提示）
RECOMMENDED_FIELDS = ["features", "notes", "equipment"]

def validate_room(room: Dict) -> Dict:
    """驗證單個會議室資料"""
    issues = {
        "errors": [],    # 必要欄位缺失
        "warnings": [],  # 重要欄位缺失
        "infos": []      # 建議欄位缺失
    }
    
    # 檢查必要欄位
    for field in REQUIRED_FIELDS:
        if field not in room or not room[field]:
            issues["errors"].append(f"缺少必要欄位: {field}")
    
    # 檢查重要欄位
    for field in IMPORTANT_FIELDS:
        if field not in room or not room[field]:
            issues["warnings"].append(f"缺少重要欄位: {field}")
    
    # 檢查建議欄位
    for field in RECOMMENDED_FIELDS:
        if field not in room or not room[field]:
            issues["infos"].append(f"缺少建議欄位: {field}")
    
    # 檢查照片
    if "images" in room:
        if not room["images"] or len(room["images"]) == 0:
            issues["warnings"].append("照片欄位為空")
    
    return issues

def validate_venue(venue: Dict) -> Dict:
    """驗證單個場地的所有會議室"""
    venue_id = venue.get("id")
    venue_name = venue.get("name")
    rooms = venue.get("rooms", [])
    
    results = {
        "venue_id": venue_id,
        "venue_name": venue_name,
        "room_count": len(rooms),
        "issues": {}
    }
    
    for room in rooms:
        room_id = room.get("id", room.get("name"))
        issues = validate_room(room)
        if any(issues.values()):  # 如果有任何問題
            results["issues"][room_id] = issues
    
    return results

def main():
    """主程式"""
    with open("venues.json", "r", encoding="utf-8") as f:
        venues = json.load(f)
    
    print("=== 會議室資料驗證報告 ===\n")
    
    error_count = 0
    warning_count = 0
    
    for venue in venues:
        result = validate_venue(venue)
        if result["issues"]:
            print(f"\n場地: {result['venue_name']} (ID: {result['venue_id']})")
            print(f"會議室數量: {result['room_count']}")
            
            for room_id, issues in result["issues"].items():
                if issues["errors"]:
                    print(f"  ❌ {room_id}: {', '.join(issues['errors'])}")
                    error_count += len(issues["errors"])
                if issues["warnings"]:
                    print(f"  ⚠️  {room_id}: {', '.join(issues['warnings'])}")
                    warning_count += len(issues["warnings"])
    
    print(f"\n=== 總計 ===")
    print(f"錯誤: {error_count}")
    print(f"警告: {warning_count}")
    
    if error_count > 0:
        print("\n❌ 資料驗證失敗！請修復錯誤後再提交。")
        return 1
    else:
        print("\n✅ 資料驗證通過！")
        return 0

if __name__ == "__main__":
    exit(main())
```

#### ✅ 改進批量更新腳本

**建議**：修改所有批量更新腳本，使用合併更新策略

```python
def merge_room_data(existing_room: Dict, update_data: Dict) -> Dict:
    """合併會議室資料（保留原有資料）"""
    merged = existing_room.copy()
    
    for key, value in update_data.items():
        # 只更新非空值
        if value is not None and value != [] and value != "":
            merged[key] = value
    
    return merged

def update_venue_rooms(venue: Dict, rooms_update: List[Dict]) -> Dict:
    """更新場地的會議室資料（合併策略）"""
    existing_rooms = {r.get("id"): r for r in venue.get("rooms", [])}
    
    updated_rooms = []
    for room_update in rooms_update:
        room_id = room_update.get("id")
        
        if room_id in existing_rooms:
            # 合併現有資料
            merged = merge_room_data(existing_rooms[room_id], room_update)
            updated_rooms.append(merged)
        else:
            # 新增會議室
            updated_rooms.append(room_update)
    
    venue["rooms"] = updated_rooms
    return venue
```

#### ✅ 建立批量更新檢查清單

**建議**：在執行批量更新前，必須完成以下檢查

```markdown
# 批量更新檢查清單

## 更新前（Pre-Update）
- [ ] 備份現有資料（`cp venues.json venues.json.backup`）
- [ ] 在小範圍測試（1-2 個場地）
- [ ] 檢查測試結果，確認無誤
- [ ] 記錄預計變更的場地數量和會議室數量

## 更新中（During Update）
- [ ] 執行更新腳本
- [ ] 監控執行過程，注意錯誤訊息
- [ ] 記錄實際變更的數量

## 更新後（Post-Update）
- [ ] 執行資料驗證腳本（`validate_room_data.py`）
- [ ] 檢查是否遺失重要欄位
- [ ] 檢查照片是否被清空
- [ ] 對比更新前後的資料差異
- [ ] 如果有問題，立即回滾（`git checkout -- venues.json`）
- [ ] 確認無誤後，才提交 commit

## Commit 規範
- Commit message 必須包含：
  - 更新的場地數量
  - 更新的會議室數量
  - 主要變更內容
  - 使用的腳本名稱
```

## Review 流程
1. 提交 PR（Pull Request）
2. 等待 Reviewer 審核
3. 修改 Reviewer 指出的問題
4. 獲得批准後才能合併

## 完整性（Completeness）

- 價格是否為最新（lastUpdated 在 3 個月內）

- 容納人數是否合理（例如：area / 1.5 ≈ capacity.theater）

- 所有會議室的欄位結構是否一致
- ID 格式是否一致（venue_id-room_number）
- 價格格式是否一致

## 目標
- 完整性 ≥ 90%
- 準確性 ≥ 95%
- 一致性 = 100%

### 核心問題

1. **批量更新腳本設計缺陷**：使用完全替換而非合併更新

### 根本原因

- **技術**：腳本設計不當
- **流程**：缺乏驗證和測試
- **組織**：缺乏培訓和規範

### 解決方案

- **立即**：修復台北喜來登資料
- **短期**：建立驗證機制和改進腳本
- **長期**：建立完整的資料管理體系

### 關鍵教訓

> **「快速迭代不能犧牲資料品質」**

批量更新雖然快速，但如果不謹慎，會造成嚴重的資料損失。必須建立完善的驗證機制，確保每次更新都是「改進」而非「破壞」。

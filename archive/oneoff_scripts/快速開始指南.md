## 一分鐘快速上手

### 1. 準備工作
```bash

# 檢查 venues.json 格式
```

### 2. 開始蒐集資料

**步驟 1**: 複製資料蒐集範本
```bash
cp data_collection_template.json data/1103_marriott.json
```

**步驟 2**: 造訪飯店官網蒐集尺寸資料
- 台北萬豪: https://www.taipeimarriott.com.tw/
- 尋找「會議與活動」或「宴會廳」頁面
- 記錄每間會議室的長、寬、高

**步驟 3**: 填寫資料蒐集檔案
```json
{
  "venueId": "1103",
  "venueName": "台北萬豪酒店",
  "rooms": [
    {
      "name": "春廳",
      "length": 16.5,
      "width": 16,
      "height": 3.5,
      "area": 80,
      "source": "官方"
    }
  ]
}
```

### 3. 轉換並更新資料

**步驟 1**: 備份原始檔案
```bash
cp venues.json venues.backup.$(date +%Y%m%d).json
```

**步驟 2**: 執行轉換腳本
```bash
node scripts/convert_dimensions_to_json.js update data/1103_marriott.json
```

**步驟 3**: 驗證更新結果
```bash
node scripts/convert_dimensions_to_json.js validate 1103
```

## 尺寸推算速查表

### 常見面積推算參考

| 坪數 | 平方公尺 | 推算尺寸（長x寬）| 層高 |
|------|---------|-----------------|------|
| 30坪 | 99 sqm | 11x9m | 3.5m |
| 50坪 | 165 sqm | 14.5x11.5m | 3.5-4m |
| 80坪 | 264 sqm | 18x14.5m | 3.5-4m |
| 100坪 | 330 sqm | 20x16.5m | 4-6m |
| 150坪 | 495 sqm | 24.5x20m | 4-6m |
| 200坪 | 660 sqm | 28.5x23m | 6-8m |
| 300坪 | 990 sqm | 35x28m | 6-8m |

### 推算公式

```javascript
// 長方形（假設長寬比 1.3:1）
長度 = sqrt(平方公尺 × 1.3)
寬度 = 平方公尺 / 長度

// 圓形
直徑 = sqrt(平方公尺 × 4 / π)

// 不規則形狀
取最接近的長方形尺寸
```

## 常見問題

### Q: 官網沒有尺寸資料怎麼辦？
A: 根據坪數推算，標註 `source: "推算"`，並在 `notes` 說明依據。

### Q: 層高不知道怎麼辦？
A: 參考以下標準：
- 一般會議室：3.5m
- 宴會廳：4-6m
- 大型宴會廳：6-8m
- 圓頂/挑高：8-12m

### Q: 面積單位不一致怎麼辦？
A: 統一使用「坪」作為 `areaUnit`，並計算 `sqm` 欄位（1坪 = 3.305785平方公尺）。

### Q: 會議室名稱有中文和英文，怎麼處理？
A: 同時填寫 `name`（中文）和 `nameEn`（英文）欄位。

### Q: dimensions 格式是什麼？
A: 統一格式為 `"LxWxHm"`，例如：`"16.5x16x3.5m"`
